# Colony & Conquest — ориентир для агента и команды

## Назначение

Этот файл — точка входа: где лежат правила, навыки, канбан и спецификации, и как вести разработку без потери контекста.

## Источники истины

| Область | Путь |
|--------|------|
| Мастер-спецификация (видение, интеграция систем) | `spec/COLONY_CONQUEST_MASTER_SPECIFICATION.md` |
| Порядок чтения спеков и карта §3 → файлы | `spec/README.md` |
| Подсистемы (детализация) | `spec/*.md` |
| Технический стек и ECS | `spec/technical_architecture_specification.md` |
| ADR / журнал решений | `docs/decisions/` (`000-template.md`, нумерованные записи; см. индекс в `docs/decisions/README.md`) |
| Структура репозитория (код, ассеты, инструменты) | `dev/PROJECT_STRUCTURE.md` |
| Ход работ (бэклог и поток) | `dev/kanban.json` (машиночитаемо), `dev/kanban.md` (человекочитаемо) |
| Детализация задачи (цель, DoD, риски) | `dev/TASK_CARD_TEMPLATE.md` → `dev/tasks/<id>.md` |
| Привязка бэклога к спекам (правила и таблица) | `dev/BACKLOG_SPEC_INDEX.md` |
| Индекс спецификаций (автогенерация) | `dev/spec_index.json`, `dev/spec_index.md` |
| Кросс-ревью спеков (отчёт) | `dev/spec_cross_review_report.md` |

## Рабочий цикл агента

1. Перед крупной задачей: прочитать `dev/kanban.md` (или JSON), **`spec_refs`** карточки и соответствующие файлы в `spec/`, при необходимости `dev/tasks/<id>.md` и ADR в `docs/decisions/`.
2. При переводе карточки в **ready** / **review**: заполнить или создать `dev/tasks/<id>.md` по шаблону, проставить `task_file`, **`spec_refs`** (если ещё не заданы) и при необходимости `adr_refs` в `dev/kanban.json`.
3. При изменении дизайна: сверяться с мастер-спецификацией и релевантным файлом в `spec/`; устойчивые решения — новый ADR.
4. После правок спецификаций: запустить `python tools/spec_index.py` для обновления индекса.
5. Соблюдать правила в `.cursor/rules/` и при релевантных задачах — навыки в `.cursor/skills/`.

## Стандарты реализации

- **Законченные решения:** `.cursor/rules/full-solutions-engineering.mdc` — задача закрывается цельным результатом, с проверкой и без скрытого долга (или он явно вынесен в другую карточку); спеки, ADR при необходимости, практики Unity/DOTS.

## Cursor

- **Правила:** `.cursor/rules/*.mdc` (всегда и по glob-ам).
- **Навыки проекта:** `.cursor/skills/*/SKILL.md` — подключайте при работе с канбаном и анализом спецификаций.
