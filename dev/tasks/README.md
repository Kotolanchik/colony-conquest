# Файлы задач (`dev/tasks/`)

Один markdown-файл на карточку канбана: имя = **`id`** из `dev/kanban.json` (например `init-unity-dots.md`).

## Когда создавать

- Перевод карточки в колонку **ready** или **review** — скопировать `dev/TASK_CARD_TEMPLATE.md` → `dev/tasks/<id>.md` и заполнить поля.
- Если задача тривиальная, допустимо оставить только `task_file` в JSON пустым, но для фич и эпиков файл обязателен.

## Отслеживаемость

```
Бэклог (kanban) → детали (tasks/<id>.md) → решения (docs/decisions/)
                      ↑
                 spec/*.md
```

При принятии архитектурного решения в ходе задачи — новый ADR в `docs/decisions/` и ссылка из файла задачи и из карточки (`adr_refs`).
