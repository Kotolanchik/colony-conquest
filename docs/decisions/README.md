# Architecture Decision Records (ADR)

Здесь фиксируются **устойчивые решения** по проекту: контекст → принятое решение → последствия. Это снижает повторное обсуждение одного и того же в чате и даёт агенту опорные точки.

## Именование

- `000-template.md` — шаблон новой записи (копировать в следующий номер).
- `NNN-kebab-case-title.md` — номер по порядку (001, 002, …), короткий slug на латинице.

## Индекс

| ADR | Тема |
|-----|------|
| 001 | ADR, канбан и отслеживаемые задачи |
| 002 | Базовая версия Unity Editor (Unity 6) для `game/Client` |

## Связь с работой

| Артефакт | Роль |
|----------|------|
| `dev/kanban.json` | Карточка задачи может ссылаться на ADR полем `adr_refs`. |
| `dev/tasks/<id>.md` | Детализация задачи (цель, спеки, DoD); при архитектурном повороте — ссылка на новый ADR. |

Новый ADR создавайте, когда решение **меняет правила игры** для кода или спеков (стек, границы модулей, сетевой контракт). Мелкие тактические выборы в рамках одной задачи — достаточно `dev/tasks/<id>.md`.
