# СОБЫТИЯ И КВЕСТЫ — ТЕХНИЧЕСКАЯ СПЕЦИФИКАЦИЯ
## Colony & Conquest — Система историй и происшествий

---

## 1. ОБЗОР СИСТЕМЫ

### 1.1 Концепция

Система событий в Colony & Conquest создаёт **эмерджентные истории** — уникальные ситуации, возникающие из взаимодействия систем, а не из заранее написанных скриптов.

> **"Истории рождаются из систем"** — как в RimWorld, каждое событие связано с состоянием мира, поселенцами и их отношениями.

### 1.2 Типы событий

| Тип | Описание | Частота |
|-----|----------|---------|
| **Случайные** | Непредсказуемые происшествия | Редко |
| **Триггерные** | Возникают при условиях | По ситуации |
| **Циклические** | Повторяющиеся (сезоны) | Регулярно |
| **Исторические** | Привязаны к эпохам | По прогрессу |
| **Персональные** | Связаны с конкретными поселенцами | Индивидуально |
| **Глобальные** | Влияют на весь мир | Очень редко |

## 2. AI DIRECTOR

### 2.1 Концепция

AI Director — система управления сложностью и драматургией, которая:
- Анализирует текущее состояние игрока
- Подбирает подходящие события
- Создаёт драматические арки (взлёты и падения)

### 2.2 Параметры состояния

```
ИЗМЕРЕНИЯ СОСТОЯНИЯ:

┌─────────────────────────────────────────────────────────────────────────────┐
│  1. БЛАГОПОЛУЧИЕ (Wealth)                                                   │
│     • Ресурсы, постройки, население                                         │
│     • 0 = катастрофа, 100 = процветание                                     │
│                                                                             │
│  2. БЕЗОПАСНОСТЬ (Security)                                                 │
│     • Армия, укрепления, отношения с соседями                               │
│     • 0 = угроза, 100 = полная безопасность                                 │
│                                                                             │
│  3. СТАБИЛЬНОСТЬ (Stability)                                                │
│     • Настроение, здоровье, социальная гармония                             │
│     • 0 = хаос, 100 = идеальный порядок                                     │
│                                                                             │
│  4. ПРОГРЕСС (Progress)                                                     │
│     • Технологии, эпоха, развитие                                           │
│     • 0 = отсталые, 100 = передовые                                         │
│                                                                             │
│  5. ИНТРИГА (Tension)                                                       │
│     • Драматичность последних событий                                       │
│     • 0 = скучно, 100 = слишком напряжённо                                  │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.3 Алгоритм выбора событий

```
ВЫБОР СОБЫТИЯ AI DIRECTOR:

1. ОЦЕНИТЬ СОСТОЯНИЕ
   wealth = calculate_wealth()
   security = calculate_security()
   stability = calculate_stability()
   progress = calculate_progress()
   tension = calculate_tension()

2. ОПРЕДЕЛИТЬ НЕОБХОДИМОСТЬ
   ЕСЛИ tension > 80:
       Нужно событие-облегчение (спасение, подарок)
   
   ЕСЛИ wealth > 80 И tension < 30:
       Нужно вызов (рейд, катастрофа)
   
   ЕСЛИ stability < 30:
       Нужно событие-стабилизация (лидерство, помощь)
   
   ЕСЛИ security < 30:
       Нужно военное событие (атака, защита)

3. ВЫБРАТЬ ПОДХОДЯЩЕЕ СОБЫТИЕ
   candidates = filter_events_by_conditions(current_state)
   weights = calculate_weights(candidates, current_state)
   selected = weighted_random_choice(candidates, weights)

4. ЗАПУСТИТЬ СОБЫТИЕ
   execute_event(selected)
   record_for_story_arc(selected)
```

## 4. ПЕРСОНАЛЬНЫЕ ИСТОРИИ

### 4.1 Система историй поселенцев

Каждый поселенец может стать героем уникальной истории:

```
ПРИМЕР ПЕРСОНАЛЬНОЙ ИСТОРИИ:

ИМЯ: Сержант Василий Петров
ЧЕРТЫ: Храбрый, Стойкий, Привязанный к товарищам

ИСТОРИЯ:
[День 1] Василий прибыл в колонию как беженец
[День 15] Спас товарища от медведя → получил прозвище "Медвежатник"
[День 45] Влюбился в медсестру Анну
[День 67] Во время рейда удерживал переправу 3 часа в одиночку
         → Повышен до сержанта
[День 89] Помолвлен с Анной
[День 120] Погиб при штурме, прикрывая отступление роты
         → Посмертно награждён, улица названа его именем

ЭФФЕКТ: +10 морали всех кто его знал на 30 дней
```

### 4.2 Архетипы историй

| Архетип | Описание | Примеры |
|---------|----------|---------|
| **Герой** | Спасает других | Спасение из пожара, медаль за отвагу |
| **Злодей** | Причиняет вред | Предательство, серийный убийца |
| **Жертва** | Страдает несправедливо | Несправедливое обвинение, болезнь |
| **Мудрец** | Приносит знания | Открытие, обучение других |
| **Влюблённые** | Романтическая история | Запретная любовь, трагедия |
| **Возрождение** | Падение и подъём | Банкротство → успех, инвалид → герой |

## 5. КВЕСТЫ

### 5.1 Типы квестов

| Тип | Описание | Пример |
|-----|----------|--------|
| **Основной** | Сюжетная линия | Развитие от колонии до империи |
| **Сторонний** | Дополнительные | Помочь торговцу, найти артефакт |
| **Персональный** | Для поселенца | Месть за убитого друга |
| **Фракционный** | Для фракции | Захватить провинцию |
| **Глобальный** | Мировой масштаб | Остановить чуму |
| **Процедурный** | Сгенерированный | Случайная комбинация условий |

### 5.2 Структура квеста

```
КВЕСТ: "Пропавший торговец"

ТИП: Сторонний
СЛОЖНОСТЬ: Средняя
НАГРАДА: 500 золота, +20 отношений с торговцами

ЭТАПЫ:
1. [НАЧАЛО] Торговец не вернулся из поездки
   └─► Поговорить с женой торговца
   
2. [РАССЛЕДОВАНИЕ] Найти следы
   └─► Отправить разведчика по маршруту
   └─► Найдена разбитая повозка
   
3. [ВЫБОР] Что произошло?
   ├─► [A] Нападение бандитов → Бой с бандитами
   ├─► [B] Нападение зверей → Поиск в лесу
   └─► [C] Предательство → Расследование в городе
   
4. [РАЗВЯЗКА] Спасти/найти торговца
   └─► Вернуть торговца (или его тело)
   
5. [НАГРАДА] Получить награду
   └─► Деньги + репутация

ВЕТВЛЕНИЯ:
• Если торговец жив: +100% награды
• Если торговец мёртв: +50% награды, похороны
• Если предательство раскрыто: новый квест на месть
```

### 5.3 Процедурная генерация квестов

```
ГЕНЕРАЦИЯ КВЕСТА:

1. ВЫБРАТЬ ШАБЛОН
   templates = ["delivery", "escort", "kill", "find", "defend", "investigate"]
   template = random_choice(templates)

2. ВЫБРАТЬ ЦЕЛЬ
   target = random_choice(available_targets)

3. ВЫБРАТЬ ЛОКАЦИЮ
   location = random_choice(discovered_locations)

4. ВЫБРАТЬ НАГРАДУ
   reward = calculate_reward(difficulty)

5. СГЕНЕРИРОВАТЬ ТЕКСТ
   description = fill_template(template, target, location, reward)

6. СОЗДАТЬ КВЕСТ
   quest = Quest(template, target, location, reward, description)
```
