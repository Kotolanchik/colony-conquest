### 1.3 Система передачи приказов

#### Механика "Потери связи"

```
ПРИКАЗ: Атаковать позицию X
    │
    ├──► Штаб (задержка: 0)
    │       └──► Армия (задержка: +5 мин)
    │               └──► Дивизия (задержка: +8 мин)
    │                       └──► Бригада (задержка: +10 мин)
    │                               └──► Батальон (задержка: +11 мин)
    │                                       └──► Рота (задержка: +11.5 мин)
    │                                               └──► Взвод (задержка: +11.8 мин)
    │                                                       └──► Отделение (задержка: +12 мин)
```

**Факторы задержки:**
- Расстояние: +1 мин за 5 км
- Радиопомехи: +50-200% задержки
- Погода (гроза, магнитная буря): +100-300%
- Уничтожение командира: +30 мин (переподчинение)
- Ночь без ПНВ: +100%

#### Каскадное подчинение при потере связи

```csharp
// Псевдокод ECS компонента командования
struct CommandHierarchy : IComponentData
{
    public Entity Superior;      // Непосредственный командир
    public Entity Subordinates;  // Подчинённые (ссылка на первого в цепочке)
    public int CommandLevel;     // Уровень в иерархии (0-9)
    public float CommandRadius;  // Радиус эффективного командования
    public float ReactionTime;   // Время реакции в секундах
    public bool HasRadio;        // Наличие радиосвязи
    public float MoraleBonus;    // Бонус к морали от командира
}

// Компонент приказа
struct Order : IComponentData
{
    public OrderType Type;       // Тип приказа
    public float3 TargetPosition;
    public Entity TargetEntity;
    public float Priority;       // 0.0-1.0
    public double IssueTime;     // Время выдачи
    public double ExpireTime;    // Время истечения
    public Entity IssuedBy;      // Кто выдал приказ
}
```

### 1.4 Мораль и командование

**Формула морали отделения:**
```
Morale = BaseMorale (100)
    + CommanderBonus (0-30)
    + VeteranBonus (0-20)
    + SupplyBonus (0-15)
    - CasualtyPenalty (0-50)
    - IsolationPenalty (0-25)
    - SuppressionPenalty (0-40)
    + CoverBonus (0-10)
    - NightPenalty (0-10)
```

**Эффекты морали:**

| Мораль | Состояние | Эффекты |
|--------|-----------|---------|
| 90-100 | Отличная | +15% точность, +10% скорость, игнор ранений |
| 70-89 | Хорошая | +5% точность, нормальная эффективность |
| 50-69 | Нормальная | Базовые параметры |
| 30-49 | Плохая | -15% точность, -10% скорость, шанс отступления 5% |
| 10-29 | Критическая | -30% точность, -25% скорость, шанс отступления 25% |
| 0-9 | Паника | -50% все параметры, шанс бегства 60% |

### 3.2 Механика укрытий

#### Формула эффективности укрытия

```
EffectiveProtection = BaseProtection 
    × QualityMultiplier 
    × OccupantState 
    × DirectionFactor

Где:
- BaseProtection: Базовый бонус укрытия (0.0-0.95)
- QualityMultiplier: 0.5-1.5 (зависит от качества постройки)
- OccupantState: 0.0-1.0 (0 = стоя, 0.5 = сидя, 1.0 = лёжа)
- DirectionFactor: 0.0-1.0 (зависит от направления атаки)
```

#### Пример расчёта

```
Солдат в окопе (BaseProtection = 0.70):
- Качество: Хорошее (1.0)
- Поза: Лёжа (1.0)
- Направление: Фронт (1.0)

EffectiveProtection = 0.70 × 1.0 × 1.0 × 1.0 = 0.70 (70%)

Урон от пули (50):
EffectiveDamage = 50 × (1 - 0.70) = 15
```

### 4.3 Система брони

#### Пробитие брони

```
PENETRATION_CHANCE = PENETRATION_VALUE / ARMOR_VALUE

Если PENETRATION_CHANCE >= 1.0:
    - Пробитие гарантировано
    - Урон = BASE_DAMAGE
    
Если 0.5 <= PENETRATION_CHANCE < 1.0:
    - Шанс пробития = PENETRATION_CHANCE × 100%
    - При пробитии: Урон = BASE_DAMAGE × 0.8
    - При непробитии: Урон = BASE_DAMAGE × 0.1 (контузия)
    
Если PENETRATION_CHANCE < 0.5:
    - Шанс пробития = PENETRATION_CHANCE × 50%
    - При пробитии: Урон = BASE_DAMAGE × 0.5
    - При непробитии: Урон = 0 (рикошет)
```

### 10.2 Мета-юниты

#### Концепция

```
МЕТА-ЮНИТ = Группа юнитов, представленная как один объект

Применение:
- Дистанция > 1000м от камеры
- Группы > 10 юнитов
- Не в активном бою

Параметры мета-юнита:
- Позиция (центр масс)
- Состав (типы и количество)
- Состояние (движение, бой, отдых)
- Здоровье (среднее)
- Боеприпасы (среднее)
- Мораль (средняя)
```

#### Переход мета-юнит ↔ индивидуальные юниты

```
РАСКРЫТИЕ МЕТА-ЮНИТА (при приближении камеры):
1. Удалить мета-юнит
2. Создать индивидуальные юниты
3. Распределить позиции вокруг центра
4. Сохранить параметры (здоровье, боеприпасы)
5. Активировать ИИ

СВОРАЧИВАНИЕ В МЕТА-ЮНИТ (при отдалении):
1. Собрать данные всех юнитов
2. Создать мета-юнит
3. Удалить индивидуальные юниты
4. Сохранить агрегированные параметры
```
