# Структура репозитория Colony & Conquest

Цель — разделить дизайн-документы, процесс разработки, журнал решений, исходный код игры и вспомогательные скрипты.

```
colony-conquest/
├── AGENTS.md                 # Точка входа для агента и команды
├── .cursor/
│   ├── rules/                # Правила Cursor (*.mdc)
│   └── skills/               # Навыки проекта (SKILL.md)
├── docs/
│   └── decisions/            # ADR: контекст — решение — последствия
├── spec/                     # Игровые и технические спецификации (источник дизайна)
├── dev/                      # Канбан, задачи, индексы спеков, служебные артефакты процесса
│   ├── tasks/                # dev/tasks/<id>.md — детализация карточки канбана
│   └── TASK_CARD_TEMPLATE.md
├── tools/                    # Скрипты (Python и др.) без зависимости от движка
└── game/
    └── Client/               # Unity (DOTS/URP): Assets/_Project/Scenes, Scripts; см. README и ProjectVersion.txt
```

## Каталоги

| Путь | Назначение |
|------|------------|
| `spec/` | Мастер-спецификация и подсистемы; не смешивать с кодом. Введение: `spec/README.md`. |
| `docs/decisions/` | ADR; связь с задачами через `adr_refs` в `dev/kanban.json`. |
| `dev/` | Канбан, шаблон и файлы задач, `spec_index.*`, этот файл. |
| `tools/` | Утилиты репозитория (индекс спеков, будущая валидация схем). |
| `game/Client/` | Корень Unity-проекта (`Assets/`, `Packages/`, `ProjectSettings/`). |

## Соглашения на будущее (код)

- **ECS/DOTS:** сборки, системы и ассеты — по структуре из `spec/technical_architecture_specification.md` (при появлении кода уточнить под фактические asmdef).
- **Сеть и симуляция:** не дублировать логику; границы — в спеках, ADR и в коде через чёткие assembly boundaries.

Обновляй этот файл при добавлении новых корневых папок (`server/`, `ci/` и т.д.).
