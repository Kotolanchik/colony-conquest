## Критерии готовности (Definition of Done)

**Законченность:** результат по задаче полный в рамках цели — не «частично сделано» без отдельной карточки на остаток; проверка и сопровождение (README / шаги) позволяют убедиться, что всё сделано.

Отметьте при закрытии:

- [ ] Требования отражены в нужных файлах `spec/` (если задача про дизайн).
- [ ] `python tools/spec_index.py` выполнен после изменений в `spec/`.
- [ ] Код/сцены соответствуют спекам и ADR (когда репозиторий содержит код — тесты или ручная проверка сценария из задачи).
- [ ] Канбан обновлён: карточка в **done**, дата в `dev/kanban.json` → `updated`.

Дополнительно по типу задачи:

| Тип | DoD |
|-----|-----|
| Только процесс / документация | Спеки или ADR обновлены; индекс спеков актуален. |
| Код | Сборка без ошибок; новые/изменённые сценарии покрыты тестами или явно помечены как tech debt с карточкой. |
