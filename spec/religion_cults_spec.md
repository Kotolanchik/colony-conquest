## 1. ОБЗОР СИСТЕМЫ

### 1.1 Концепция

Религиозная система добавляет **духовное измерение** в игру:
- Разные религии с уникальными учениями
- Влияние веры на поведение поселенцев
- Конфликты между конфессиями
- Священные войны и крестовые походы

| Уровень | Вера | Эффекты |
|---------|------|---------|
| **0-20** | Атеист | Нет бонусов, +10% научный прогресс |

| **41-60** | Верующий | +10% мораль, +5% стабильность |

| **81-100** | Фанатик | +20% мораль, +15% стабильность, +10% продуктивность, риск радикализации |

## 7. ТЕХНИЧЕСКАЯ МОДЕЛЬ ДАННЫХ (ECS)

### 7.1 Компоненты

| Компонент | Назначение | Ключевые поля |
|-----------|------------|---------------|
| `ReligionAffiliation` | Конфессиональная принадлежность сущности | `ReligionId`, `SectId`, `IsSecular` |
| `FaithState` | Текущее состояние веры поселенца | `FaithLevel` (0..100), `Fanaticism` (0..100), `Doubt` (0..100) |
| `ReligiousInfluenceSource` | Источник влияния (храм, проповедник, культ-лидер) | `InfluencePower`, `Radius`, `DoctrineTag` |
| `ReligiousTensionState` | Индекс напряжённости между группами | `TensionScore`, `RecentIncidents`, `SuppressionLevel` |
| `HolySiteData` | Данные святого места | `ReligionId`, `PilgrimageValue`, `StrategicValue` |
| `HolyWarIntent` | Намерение фракции начать священную войну | `TargetFaction`, `CasusBelli`, `Preparedness`, `DeadlineTick` |
| `CultActivityState` | Активность деструктивного культа | `RecruitmentRate`, `HiddenCells`, `RadicalizationRisk` |

### 7.2 Инварианты и ограничения

- `FaithLevel`, `Fanaticism`, `Doubt` ограничены диапазоном `0..100`.
- Секта не может быть назначена без валидного `ReligionId`.
- `HolyWarIntent` создаётся только при `CasusBelli` из белого списка: святое место, преследование верующих, разорванный договор.
- При `RadicalizationRisk >= 70` обязателен тик системы правопорядка (см. `spec/crime_justice_spec.md`).

## 8. ДИНАМИКА ВЕРЫ И КОНФЛИКТА

### 8.1 Формула изменения веры (суточный тик)

`FaithDelta = TempleVisit * 1.0 + SermonExposure * 0.8 + PersonalCrisis * 0.6 + CommunityCohesion * 0.4 - EducationPressure * 0.5 - ProsperitySecularPull * 0.3`

`FaithLevel_next = clamp(FaithLevel_current + FaithDelta, 0, 100)`

### 8.2 Вероятность конверсии

`ConversionChance = clamp((InfluencePower + SocialPressure + BenefitExpectation - Resistance) / 100, 0.01, 0.85)`

Где:
- `InfluencePower` зависит от храмов, проповедников и авторитета духовенства.
- `Resistance` повышается образованием, стабильностью и лояльностью к исходной вере.

### 8.3 Индекс межконфессионального напряжения

`TensionScore = IncidentWeight + IdeologyDistance * 0.6 + ResourceCompetition * 0.5 + PropagandaPressure * 0.4 - LawEnforcement * 0.7 - DialoguePrograms * 0.5`

Пороговые зоны:
- `0..30`: управляемое напряжение.
- `31..60`: локальные инциденты.
- `61..80`: массовые беспорядки/погромы.
- `81..100`: высокий риск священной войны или раскола фракции.

## 9. ПАЙПЛАЙН СВЯЩЕННЫХ ВОЙН

### 9.1 Машина состояний

1. **TriggerDetected** — зафиксирован `CasusBelli`.
2. **DoctrineApproval** — проверка внутренней доктрины и поддержки элит.
3. **UltimatumPhase** — дипломатический ультиматум цели.
4. **Mobilization** — подготовка армии, усиление пропаганды, рост морали.
5. **HolyWarActive** — активная фаза конфликта.
6. **Resolution** — мир, аннексия, конверсия территории или отступление.

### 9.2 Системные события

- `HolyWarDeclaredEvent`
- `HolySiteCapturedEvent`
- `ForcedConversionEvent`
- `FaithCrisisEvent`
- `ReligiousPeaceTreatyEvent`

События публикуются в общий пайплайн `spec/events_quests_spec.md`.

## 10. ПОВЕДЕНИЕ AI-ФРАКЦИЙ

### 10.1 Приоритеты религиозной политики (веса)

| Профиль фракции | Миссионерство | Терпимость | Силовое давление | Секуляризация |
|-----------------|---------------|------------|------------------|---------------|
| Теократия | 0.9 | 0.2 | 0.7 | 0.0 |
| Монархия | 0.5 | 0.5 | 0.4 | 0.2 |
| Республика | 0.3 | 0.8 | 0.2 | 0.6 |
| Технократия | 0.1 | 0.7 | 0.1 | 0.9 |

### 10.2 Ограничения AI

- AI не инициирует священную войну при низкой боеготовности (`Preparedness < 50`).
- При внутреннем `TensionScore > 70` приоритет сдвигается в подавление беспорядков.
- Фракции с доктриной терпимости штрафуются за агрессивную конверсию (дипломатический урон).

## 11. ПРОВЕРКА И TELEMETRY

### 11.1 Acceptance-сценарии

1. Поселенец с высоким `EducationPressure` теряет веру быстрее при равных прочих.
2. При захвате святого места запускается корректный пакет событий и изменение морали.
3. При `TensionScore >= 81` система создаёт кризисный сценарий (беспорядки/раскол/война).
4. Деструктивный культ повышает `RadicalizationRisk` и влияет на криминальную статистику.
5. Мирный договор после священной войны снижает напряжение и фиксируется в дипломатии.

### 11.2 Метрики аналитики

| Метрика | Тип | Назначение |
|---------|-----|------------|
| `faith_level_avg` | Gauge | Средний уровень веры по фракции |
| `conversion_events_total` | Counter | Количество конверсий за период |
| `religious_tension_avg` | Gauge | Среднее межконфессиональное напряжение |
| `holy_wars_started_total` | Counter | Количество объявленных священных войн |
| `cult_radicalization_cases` | Counter | Инциденты радикализации культов |
